# 产品需求文档 (PRD) - Markdown 转公众号排版工具 V1.0

## 1. 文档信息
- **产品名称**：MD2WeChat（暂定，可在开发时确定最终名）
- **版本**：V1.0
- **创建日期**：2026-05-16
- **产品形态**：纯前端 Web 单页应用（SPA）
- **核心价值**：Markdown 写作 → 实时预览 → 10 套公众号风格主题 + 暗黑模式适配 → 一键复制富文本，直接粘贴进微信公众平台后台即可发布。

---

## 2. 产品概述
### 2.1 产品定位
面向公众号创作者的专业排版效率工具。它让用户用 Markdown 快速写作，并实时生成**完全符合微信公众号后台 HTML 规范、带有内联样式的富文本**。内置 10 套精美主题（包含暗黑模式），一键复制后可直接在公众号编辑器发布，无需任何二次调整。

### 2.2 目标用户
- 用 Markdown 写作的技术博主、教程作者
- 内容运营与小编，希望统一排版风格
- 有轻度公式/脚注/视频需求的严肃内容创作者

### 2.3 核心痛点解决
- 省去在微信后台反复调整字体、颜色、间距的工作
- 消除 Markdown 本地预览与微信公众号实际效果不一致的问题
- 提供专业且多变的主题，适应不同调性文章
- 解决黑暗模式下的排版失配问题

## 3. 用户故事
- **作为**技术博主，**我想**在工具里直接写 Markdown，或用 .md 文件导入，**以便**不改变本地写作习惯，快速得到公众号排版。
- **作为**运营小编，**我想**在 10 套主题之间快速切换，并预览暗黑模式效果，**以便**根据文章风格和读者阅读环境选择最合适的主题。
- **作为**有复杂内容需求的创作者，**我想**在文章里使用数学公式、脚注、表格和视频卡片，**以便**我的文章在公众号里也能保持专业感。
- **作为**频繁更新的用户，**我想**让工具自动保存我的历史文章在本地，**以便**我随时调出修改或重发，且不用担心隐私泄露。

## 4. 功能需求

### 4.1 编辑与输入（P0）
| 编号 | 功能 | 描述 |
|------|------|------|
| F1.1 | Markdown 编辑器 | 左侧提供一个功能完整的文本编辑区，支持语法高亮（可选）、行号。用户可直接输入。 |
| F1.2 | 粘贴 Markdown 源码 | 支持从系统剪贴板直接粘贴 Markdown 文本到编辑器。 |
| F1.3 | 上传 .md 文件 | 提供一个「打开文件」按钮或拖拽区域，用户可将本地 .md 文件加载到编辑器中。 |
| F1.4 | 分屏布局 | 左侧 Markdown 编辑器，右侧实时预览。分屏比例可拖动调节（默认左 40% 右 60%）。 |

### 4.2 预览与主题（P0）
| 编号 | 功能 | 描述 |
|------|------|------|
| F2.1 | 实时预览 | 右侧面板实时渲染公众号效果，更新延迟 <200ms。预览区域以手机模型框呈现，尺寸 375×812（iPhone X 比例），背景浅灰，内容区白色。 |
| F2.2 | 10 套内置主题 | 提供至少 10 套主题，覆盖不同风格：<br>「默认纯净」「文艺浅调」「深阅读」「商务简雅」「科技极简」「萌趣手账」「灰度杂志」「墨夜星辰」「森林日记」「暖阳笔记」<br>（主题名可修改，需定义每套的配色、字号、间距） |
| F2.3 | 暗黑模式适配 | 其中「墨夜星辰」为主题化的暗黑风格主题，同时每一套主题均可切换至暗色模式变体。工具提供预览暗黑模式效果的功能（模拟微信在系统暗黑模式下的显示）。用户可选择生成针对暗黑模式优化过的 HTML（使用暗色背景、浅色文字等行内样式），以提升在微信夜间模式下的阅读体验。 |
| F2.4 | 主题即时切换 | 顶部工具栏提供主题选择器，点击后实时更新右侧预览。 |

### 4.3 复制与导出（P0）
| 编号 | 功能 | 描述 |
|------|------|------|
| F3.1 | 一键复制渲染后的富文本 | 主按钮「复制内容」：将右侧预览区**已渲染的 DOM 节点**复制到剪贴板，支持粘贴到微信公众平台编辑器后样式完全保留。 |
| F3.2 | 复制 HTML 源码 | 次按钮「复制 HTML」：弹窗显示生成的行内样式 HTML 字符串，供用户手动全选复制，作为备用方案。 |

### 4.4 图片处理（V1.0 策略，P2）
- 阶段 1：所有图片使用**占位图**替代。Markdown 中的 `![alt](url)` 渲染为一个带 alt 文字的灰色占位框，提示用户需在公众号后台手动替换图片。
- 后续版本可集成图床插件，但 V1.0 不做图床上传。

### 4.5 Markdown 语法支持与特殊格式

**基础语法（P0）**
- 标题 H1-H6
- 加粗、斜体、删除线
- 无序/有序列表
- 引用块
- 链接、图片占位符
- 行内代码、代码块（需带语法高亮）
- 表格（自适应、斑马纹）
- 分割线
- 任务列表 `- [ ]` / `- [x]`

**高级扩展（P1 必须支持）**
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 代码块语法高亮 | 自动识别语言，应用主题化高亮样式，行内样式输出（例如 highlight.js + 行内样式提取）。手机端不溢出。 | P0 |
| 表格 | 支持复杂表格，带斑马纹、边框，自适应，在 375px 宽度内可滚动。 | P0 |
| 数学公式 | 支持 LaTeX 语法（`$...$` 行内和 `$$...$$` 块级），可选用 KaTeX 渲染为 HTML+CSS（内联样式），确保在微信中显示正常。 | P1 |
| 脚注 | 支持 Markdown 脚注语法 `[^1]`，渲染为底部脚注列表，并自动处理内联链接跳转（微信内可用的锚点链接）。 | P1 |
| 视频卡片 | 支持类似 `![video](url)` 的特殊语法，渲染为视频占位符卡片（封面图 + 播放按钮），引导用户在公众号后台替换为视频。或自动转为微信支持的视频链接卡片（如果平台允许）。 | P1 |

> 注：数学公式、脚注、视频卡片在 V1.0 中必须可用，但允许有最小可用实现。例如公式使用 KaTeX 本地库渲染成 span 标签；脚注生成底部列表；视频显示为带样式的占位模块。这些功能经过测试，确保在微信中粘贴后不乱。

### 4.6 本地历史记录（P1）
| 编号 | 功能 | 描述 |
|------|------|------|
| F4.1 | 自动保存 | 每次编辑内容实时保存到浏览器本地存储（IndexedDB），防止意外关闭丢失。 |
| F4.2 | 历史列表 | 侧边栏或独立视图展示保存的文章列表，包含标题（自动提取第一个 H1，或无标题）、保存时间、预览片段。 |
| F4.3 | 加载历史 | 点击历史记录条目，将 Markdown 内容加载到编辑器，并沿用上次使用的主题。 |
| F4.4 | 删除历史 | 支持删除单条或清空全部历史，数据完全在用户浏览器，不上传服务器。 |

## 5. 主题系统详细要求
- 每个主题需定义：正文字色、背景色、标题色（H1-H6）、加粗色、斜体、删除线、链接色、列表符号色、引用块左侧边框颜色与背景、代码块背景/文字/高亮色、表格边框/表头背景/斑马纹、分割线颜色、行高、段间距。
- 10 套主题中至少包含 2 套暗色背景主题（如「墨夜星辰」和另一暗色模式变体），便于暗黑模式用户。
- 主题在预览中实时切换，生成 HTML 时必须将全部样式写入行内 `style` 属性，不允许使用 class 或 id。
- 需提供预览暗黑模式的开关，让用户了解文章在微信夜间模式下的显示效果。该功能可通过给预览区叠加 CSS 滤镜或通过 `prefers-color-scheme` 模拟实现，但最终生成的 HTML 仍以所选主题的行内样式为准（暗黑模式优化版主题直接使用了深色背景等样式）。

## 6. 非功能需求
- **性能**：键入到预览更新延迟 ≤ 200ms，公式较多时 ≤ 500ms。
- **兼容性**：支持 Chrome、Edge、Safari、Firefox 最新两个大版本。
- **隐私与安全**：一切文本编辑、存储均在本地浏览器完成，无服务端交互，无数据上传。使用本地存储时需明确告知用户。
- **可访问性**：工具栏按钮有标题说明，复制操作有视觉反馈（toast 提示“已复制”）。
- **国际化**：V1.0 仅支持中文界面。

## 7. 用户界面与交互原型
**整体布局：**
- 左侧：Markdown 编辑器 (可调节宽度，默认 40%）
- 右侧：手机模型预览窗口（375px 宽，白色卡片，灰色背景）
- 顶部工具栏：
  - 最左：Logo / 工具名称
  - 左侧功能区：「打开 .md 文件」图标按钮、「历史记录」图标按钮
  - 中间：主题选择下拉（显示当前主题名，点击展开 10 个主题列表，带色块预览）
  - 右侧：暗黑模式预览开关（模拟/关闭）
  - 最右：主按钮「📋 复制内容」（绿色强调），次按钮「</> 复制 HTML」
- 历史记录面板：点击图标从左侧滑出，显示列表，可搜索标题，点击加载。
- 底部状态栏（可选）：显示字数、保存状态。

**交互细节：**
- 加载 .md 文件后，编辑器内容替换，同时清空历史保存点？不，应保留旧文档可撤销，或提示用户保存。V1.0 做简化：加载文件替换编辑器内容，并自动保存到历史。
- 复制成功后弹出 toast：“已复制，可直接去公众号后台粘贴”。
- 暗黑模式预览开关仅影响右侧预览，不影响编辑器主题，也不改变生成的 HTML 代码（除非用户选择了暗黑主题）。

## 8. 技术选型建议（供开发参考）
- **前端框架**：React / Vue（任选）
- **Markdown 解析**：`markdown-it` + 插件（`markdown-it-footnote`，`markdown-it-texmath` 或自行集成 KaTeX）
- **公式渲染**：`KaTeX`（轻量、快速、可输出 HTML+CSS）
- **代码高亮**：`highlight.js`，并实现行内样式提取（关键难点：需将类样式转为行内 style）
- **文件读取**：FileReader API 读取 .md 文件
- **本地存储**：`idb-keyval` 或原生 IndexedDB 封装
- **复制实现**：`Clipboard API` + 降级方案 `execCommand`，复制渲染后 DOM 节点时使用 `cloneNode` + 包装 `<div>` 放入剪切板
- **主题管理**：定义 CSS-in-JS 对象，每个主题包含完整的样式映射，渲染时直接应用到元素行内样式。

## 9. 里程碑与发布计划
1. **原型阶段（2周）**：完成基础 Markdown 解析、一套默认主题、分屏预览，不含复制和历史。
2. **MVP V1.0（4周）**：
   - 10 套主题 + 暗黑模式预览开关
   - 基础复制功能
   - 代码高亮、表格、脚注、公式（最小可用）
   - 视频占位符
   - 本地历史（自动保存、列表、加载）
   - 文件上传
   - 内部测试与样式兼容性验证（在真实微信后台粘贴测试）
3. **V1.1（后续迭代）**：优化公式和视频展示、增加主题自定义、导出 HTML 增强、支持更多语法（如定义列表、缩写）。

## 10. 成功指标（V1.0）
- 完成一次“写 Markdown → 选主题 → 复制 → 粘贴进公众号”全流程平均时长 < 1 分钟。
- 在 10 个主流主题下，粘贴到微信公众平台后，在 iOS/Android 客户端和黑暗模式下，格式错乱率 < 3%。
- 本地历史功能成功保存≥100 篇文章，且加载速度 < 500ms。
- 上线后 2 个月内，周活跃用户数 ≥ 1000（若配合推广）。
